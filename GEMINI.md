# GEMINI.md - 專案規範與備忘錄

此檔案用於記錄本專案的關鍵開發規範與系統限制。每次對專案進行開發或變更時，AI 助理必須重新載入此檔案以遵守以下規範：

## 1. 開發基本規範
* **語言規範**：所有的說明、日誌、文件、程式碼註解及對使用者的回覆，必須統一使用**繁體中文**。
* **終端命令規範**：若使用 PowerShell，必須調用系統中最新的 **PowerShell 7.x (pwsh)** 版本，以確保絕佳的 UTF-8 相容性，減少因舊版 Windows PowerShell 造成的編碼混亂。
* **檔案編碼規範**：本專案所有的檔案（包括 HTML、CSS、JS、MD、LOG 等）都必須以 **UTF-8 加上 BOM** 的格式寫入，且檔案開頭不能包含字串 "\uFEFF"（必須直接以 BOM 的二進位位元組 `EF BB BF` 寫入）。

## 2. 檔案寫入安全與備份機制
* 每次儲存或修改任何程式檔案或文件時，為避免因突然中斷造成 0 byte 檔案毀損或亂碼，應採取以下步驟：
  1. 建立 `.bak` 備份檔案。
  2. 寫入加入 BOM 的內容。
  3. 驗證新寫入的檔案存在、大小大於 0、且開頭三個位元組為 `EF BB BF`。
  4. 驗證通過後才移除 `.bak` 備份檔；驗證失敗則立即還原備份。
* 可以使用本專案的工具腳本來執行安全的寫入作業：
  ```bash
  node scripts/saveHelper.js write <目標檔案> <內容暫存檔>
  ```

## 3. Git 自動提交與推送
* 每次進行任何檔案變更、功能新增、或版本更新完成後，必須自動執行以下 Git 指令：
  1. `git add .`
  2. `git commit -m "本次變更簡短摘要（使用繁體中文）"`
  3. `git push origin main`
* Git 的遠端儲存庫網址為：`https://github.com/Flash2u/Sudoku.git`
* 主分支名稱為 `main`。

## 4. 版本控制與變更日誌 (`modify.log`)
* **版本號碼**：
  * 從 **`1.0`** 開始。
  * 每次功能修改或版本更新，版號**遞增 `0.1`**。
* **玩法說明整合**：
  * 每次更新版號時，必須同步在 [index.html](file:///C:/Users/yuki/OneDrive/開發/Sub_Sudoku/index.html) 的「如何玩數獨（`#modal-help`）」視窗中，更新顯示當前的**版本號碼**以及**製作日期（格式如 YYYY-MM-DD）**。
* **變更日誌 (`modify.log`)**：
  * 每次版本異動，必須將變更寫入位於專案根目錄的 `modify.log`。
  * `modify.log` 必須使用 **繁體中文**，編碼格式為 **UTF-8 加 BOM**。
  * 格式必須將**最新版本寫在最上面**，每個版本包含以下資訊：
    1. 版本號碼
    2. 製作日期
    3. 本次版本的變更內容
  * 可使用工具腳本來安全更新日誌：
    ```bash
    node scripts/saveHelper.js updateLog <版本號碼> <日期> "變更內容第一行\n變更內容第二行"
    ```
